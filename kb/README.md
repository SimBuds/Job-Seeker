# Knowledge base

Everything in `kb/` is git-tracked, model-portable, and **read-only at runtime**.
The CLI loads it; only `jobhunt convert-resume` regenerates `profile/`, and only
humans edit `policies/`.

## Subdirectories

- `profile/` — generated by `jobhunt convert-resume` from
  `Casey_Hsu_Resume_Baseline.docx`. Source of truth for tailoring.
  - `verified.json` — the structured snapshot. Tailoring prompts are constrained
    to facts in this file (the no-fabrication rule).
  - `resume.md`, `skills.md`, `work-history.md`, `education.md` — human-readable
    mirror of the same data.
- `policies/` — hand-edited rules.
  - `tailoring-rules.md` — prompt-injectable mirror of
    `Resume_Tailoring_Instructions.md` (root-level source of truth).

## Editing workflow

- **To update profile facts:** edit `Casey_Hsu_Resume_Baseline.docx`, then run
  `uv run jobhunt convert-resume`. Never edit `profile/*.md` by hand — they
  get overwritten.
- **To update tailoring rules:** edit `Resume_Tailoring_Instructions.md` (the
  source of truth), then sync the trimmed mirror in `policies/tailoring-rules.md`.
